# Chrome Web Store 发布指南

## 提交步骤

### 1. 访问开发者控制台
- 打开 [Chrome Developer Dashboard](https://chrome.google.com/webstore/devconsole/)
- 登录您的开发者账号

### 2. 创建新项目
- 点击"添加新项目"按钮
- 同意开发者服务条款

### 3. 上传扩展包
- 选择文件：`chrome-font-subseter-v1.0.0.zip`
- 等待上传完成
- 系统会自动验证包内容

### 4. 填写商店列表信息

#### 基本信息
- **扩展名称**：字体子集化工具
- **简短描述**：字体子集化工具 - 快速创建包含所需字符的小字体文件，优化网页加载速度
- **详细描述**：使用 `STORE_DESCRIPTION.md` 中的内容

#### 分类
- **类别**：生产力工具
- **语言**：中文（简体）

#### 图标
- 已包含在zip包中（16x16, 32x32, 48x48, 128x128）

### 5. 截图要求

**必需的截图尺寸：**
- 1280x800 或 640x400
- 至少1张，最多5张
- PNG或JPG格式
- 实际使用截图，不包含浏览器边框

**建议截图内容：**
1. 主界面 - 显示上传区域
2. 字符输入 - 显示快捷按钮和字符输入
3. 预览功能 - 显示字体预览效果
4. 生成结果 - 显示压缩统计信息
5. 下载选项 - 显示格式选择

### 6. 隐私和权限
- **权限**：无（我们不需要任何特殊权限）
- **隐私政策**：不需要（扩展不收集用户数据）

### 7. 发布选项
- **可见性**：公开
- **定价**：免费
- **地区**：所有地区

### 8. 提交审核
1. 点击"提交审核"
2. 等待审核（通常3-5个工作日）
3. 审核通过后会自动发布

## 常见问题

### Q: 审核被拒绝怎么办？
A: 查看拒绝原因，修改后重新提交。常见原因：
- 描述信息不准确
- 截图不清晰
- 功能不完整

### Q: 如何更新扩展？
A:
1. 修改版本号（manifest.json）
2. 重新打包
3. 在开发者控制台上传新版本

### Q: 需要多长时间通过审核？
A: 通常3-5个工作日，高峰期可能更长。

## 注意事项

1. **功能完整性**：确保所有功能都能正常工作
2. **UI/UX**：界面美观，易于使用
3. **无错误**：控制台不能有JavaScript错误
4. **描述准确**：不要夸大功能
